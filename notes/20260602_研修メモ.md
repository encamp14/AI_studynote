# AI研修メモ｜2026年6月2日

## Markdown（MD形式）
- AIが読み取りやすい形式
- クライアント情報をMD形式で共有するとメンバーがAIに読み込ませやすい

---

## NotebookLM
- YouTubeや資料を読み込ませて、その内容だけで質問に答えてくれる
- Geminiはネット全体から取ってくるが、LMはアップした内容だけ → 嘘がない
- 活用アイデア：
  - 顧客リストをLMにまとめて「表にして」と指示
  - 予約帳票を全部コピペ → 名前・メールアドレス・誕生日だけ抽出
  - 男女比率・年齢別グラフ作成・マインドマップ・スライド作成

---

## Gemini
- プロンプトの精度が大事 → 指示によってアウトプットが全然違う
- `#用途` という項目を作るとプロンプトを使い回しできる
- 提案してほしい数を指定するとしっかりその数を返してくれる

### Gemini Gem（専用プロンプト）
- 前情報を入れ込んでおける → 毎回同じ情報を入力しなくていい
- 活用アイデア：予約者帳票の自動作成Gem
  - 日付・人数・プランをコピペするだけで帳票が出てくる形に
  - GemにGemを作らせる（プロンプトをAIに考えさせる）

## Git / GitHub
| 言葉 | 意味 |
|---|---|
| リポジトリ | GitHubにあるフォルダ（写真の共有アルバム） |
| Git | PC内の履歴管理カメラ（コミット＝写真撮影） |
| GitHub | ネット上の写真共有サイト（共有金庫） |
| Push | ローカル → GitHubに送る |
| Pull | GitHub → ローカルに取ってくる |
